## AK47 加载状态误报修复（index35.html）

- 根因：`ak47Model` 异步加载，GLTFLoader 回调在 `setupController` 之后才执行，`attachAK47()` 静默失败（两个条件不同时满足）
- 修复：
  1. 新增 `ak47Attached` 布尔标志，防止重复挂载
  2. GLTFLoader 回调成功后立即调用 `attachAK47()`，确保模型加载完后能挂上去
  3. 调试面板同时显示 `模型:✅/⏳` 和 `挂载:✅/❌` 两个状态，方便区分问题阶段

## 加载卡住修复（index35.html）

- 根因：loadingEl 的隐藏依赖两个异步回调全部触发，资源加载失败或网络慢时永远卡在"加载中"
- 修复：新增 `hideLoadingOnce()` 统一函数 + 12秒兜底超时，无论加载成功/失败/超时都会关闭加载画面
- 同时更新了纹理和 GLTF 加载的错误回调，避免加载失败后仍显示 spinner

## 持续卡在加载（index35.html）

- 两个资源文件都在，但页面还是卡住——极可能是 Three.js CDN（unpkg.com）国内网络不通
- 修复：CDN 换成 jsdelivr（国内 CDN 节点），超时从12秒改为5秒
- 新增 6 秒检测：`window.__THREE_READY__` 标志，Three.js 加载失败则显示"请确保使用 HTTP 服务器访问"提示
- **关键**：本地双击打开 HTML（`file://`）会导致 CDN 请求被浏览器拦截。必须通过 HTTP 服务器访问（如 `npx serve .` 或 VSCode Live Server）
